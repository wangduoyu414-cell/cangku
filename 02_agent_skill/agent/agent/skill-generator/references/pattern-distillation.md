# pattern-distillation（规律提纯）

不要直接把原始案例抄成规则。先做提纯，再决定它该进入哪一层资产。

## 五层提纯

1. `raw case（原始案例）`  
   保留触发语句、上下文、动作、结果、返工点。

2. `shared invariant（共享不变量）`  
   只保留跨案例稳定成立的条件，不记录一次性偶然细节。

3. `default path（默认路径）`  
   把成功概率最高、成本最稳的做法写成默认流程。

4. `guardrail（护栏）`  
   把高频失败或低频高损失败升级为停止、降级、复核或切换规则。

5. `asset（资产）`  
   按内容形态把规则落到合适位置：
   - `SKILL.md`：核心流程、关键护栏、默认路径
   - `references（参考资料）`：按需知识与长说明
   - `examples（示例）`：格式样例、好输出、失败回放
   - `scripts（脚本）`：稳定重复且可验证的逻辑
   - `assets（资产）`：模板、骨架、可复用文件

## 规则升级门槛

- 重复成功：升级为 `default path（默认路径）`
- 重复失败：升级为 `guardrail（护栏）`
- 低频但高损失败：升级为 `hard stop（强阻断）`
- 只出现一次的技巧：先记为 `note（注记）`
- 只对当前模型或当前工具有效：记为 `model-sensitive hint（模型敏感提示）`

## 四类内容必须区分

- `pattern（规律）`：跨案例稳定成立
- `workaround（权宜技巧）`：仅在特定模型、工具、环境下有效
- `template（模板）`：稳定的输出骨架
- `anecdote（个例经验）`：只能参考，不能直接固化

## 防止过拟合

- 不要为了一个失败样例就扩张边界
- 不要把具体关键词当规则本身
- 不要把“当前模型做不好”误写成“永远必须这么做”
- 每条规则都记录 `source（来源）`、`confidence（置信度）`、`freshness（新鲜度）`
