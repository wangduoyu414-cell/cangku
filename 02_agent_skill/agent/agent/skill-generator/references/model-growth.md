# model-growth（模型共成长）

一个能长期存活的 `skill（技能）`，必须把内容拆成两层：

- `stable core（稳定内核）`
- `model-sensitive layer（模型敏感层）`

## stable core（稳定内核）

只放模型再强也不该省掉的东西：

- 边界定义
- 输入输出契约
- 风险阻断
- 证据要求
- 验证步骤
- 关键术语与分类

## model-sensitive layer（模型敏感层）

只放“当前模型版本下”特别有用，但未来可能应该删除的东西：

- 冗长分步提示
- 过于细颗粒度的思考展开
- 为弥补当前模型短板而加的 workaround（权宜技巧）
- 特定模型偏好的输出组织方式
- 只在当前工具行为下成立的指令顺序

## 升模时先做的五件事

1. 用同一套 `evals（评测）` 复跑新模型
2. 找出旧版 `skill（技能）` 中哪些步骤已经成为冗余脚手架
3. 保留 `stable core（稳定内核）`，优先删减 `model-sensitive layer（模型敏感层）`
4. 检查新模型是否引入新的误用模式、误触发或过度自信
5. 重新计算 `delta（增益）`，确认当前 `skill（技能）` 仍然值得存在

## 简化优先级

- 先删冗余分步提示
- 再删已经不再贡献增益的模板化话术
- 最后才考虑缩减真正的安全、契约、验证规则

## 不能删除的内容

- 安全边界
- 产物契约
- 风险升级点
- 基线对比与验证闭环

模型变强后，一个优秀 `skill（技能）` 的表现不是“指导更多”，而是“在不牺牲安全和契约的前提下，指导更少、产出更好”。
