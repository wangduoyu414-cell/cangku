# experience-management（经验管理）

把案例沉淀成 `experience ledger（经验账本）`，目标不是囤积材料，而是回答两件事：

- 哪些做法值得复用
- 哪些错误必须被阻断

## 每条经验至少记录

- `signal（信号）`：用户请求、文件类型、触发语句
- `context（上下文）`：任务约束、输入质量、依赖条件
- `action（动作）`：模型实际采用的方法
- `outcome（结果）`：成功、失败、返工、人工接管点
- `quality delta（质量差值）`：相对基线是更好、更差还是持平
- `cost（成本）`：时间、令牌、工具调用
- `rule candidate（候选规则）`：可能沉淀出的默认路径、护栏或模板
- `confidence（置信度）`：单次观察、重复观察、强证据
- `freshness（新鲜度）`：是否可能随模型、工具、规范变化而失效

## 升级规则

- 高频成功路径：升级为 `default path（默认路径）`
- 高频失败路径：升级为 `guardrail（护栏）`
- 低频但高损失败：升级为 `hard stop（强阻断）`
- 仅一次出现的技巧：先保留为 `note（注记）`
- 只在当前模型下成立：标记为 `model-sensitive hint（模型敏感提示）`

## 删除规则

以下情况不要继续保留老经验：

- 已被新模型自然消化
- 与当前工具链不再匹配
- 只能解释个例，不能泛化
- 修复了一个失败，却制造了更多误触发
