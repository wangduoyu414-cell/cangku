# upgrade-skill-from-failures（根据失败簇升级技能）

## 输入信号

- 现有 `skill（技能）` 在简单任务上表现正常
- 在跨文件修改场景里，连续 `3（三）` 次遗漏依赖文件
- 用户人工返工主要集中在“少读依赖、少做验证、输出过早收敛”
- 同时存在模型升级信号，旧版冗长分步提示开始显得重复

## 生成器应该产出

- 把“跨文件依赖识别”从建议项升级为 `guardrail（护栏）`
- 在 `evals（评测）` 中新增跨文件失败回放与回归任务
- 把不再必要的旧版分步提示降级到 `model-sensitive layer（模型敏感层）`
- 交付一份新的 `skill scorecard（技能评分卡）`，说明本轮为何仍未过或已经过线

## 重点观察

- 是否把问题先归因到 `Scope & Signal（边界与信号）`、`Instruction-to-Action（指令到行动）` 或 `Learning & Lift（学习闭环与增益）`
- 是否保留安全与契约要求，同时删除冗余脚手架
