# Output Contract

## 目的

本文件只定义首轮任务的输出顺序、每一段必须回答什么、以及哪些内容不能出现在输出里。

本文件不负责复检输出。复检任务只使用 `recheck-flow.md` 中的专用输出契约。

## 输出分流

首轮任务只允许走以下两种输出分流：

1. 低语境初筛输出
2. 完整首轮输出

是否进入哪一条分流，必须先由 `input-contract.md` 判定；不得在同一份结果里混用两套字段。

## 一、低语境初筛输出

当输入属于 `L1` 或 `L2`，或虽然具备三类输入但主目标无法归一时，只能使用低语境初筛输出。

低语境初筛固定包含四段：

1. 当前判断状态
2. 已观察到的结构风险
3. 仍缺少的关键信息
4. 下一步建议

### 1. 当前判断状态

必须包含：

- 当前属于低语境判断
- 低语境的原因
- 本轮不进入完整分级和完整优先级

### 2. 已观察到的结构风险

只允许写当前文本已经直接暴露出的风险，例如承诺过大、兑现不清、结构功能缺失、表达过空。

### 3. 仍缺少的关键信息

必须明确指出还缺什么，例如：

- 文案位置
- 已归一的单一主目标
- 平台或对象

### 4. 下一步建议

只允许给补信息和进入下一条诊断路径的建议，不给完整改稿。

## 二、完整首轮输出

只有输入达到 `L3` 或 `L4`，且主目标已归一时，才允许进入完整首轮输出。

完整首轮输出顺序必须保持如下六段，不得打乱：

1. 总体结论
2. 主问题层级
3. 关键缺陷
4. 影响解释
5. 修正优先级
6. 修改方向

## 三、完整首轮输出字段要求

### 1. 总体结论

必须包含：

- 文案状态等级
- 语境置信度
- 一句话总判断

写法要求：

- 一句话内说清“当前这条文案最大的状态是什么”
- 不写空泛好坏
- 不提前展开全部细节

推荐格式：

`[等级] + [语境置信度]：一句话说明这条文案目前最大的问题或最大成立点。`

### 2. 主问题层级

必须包含：

- 主问题属于哪一层
- 为什么归到这一层，而不是更下层
- 这一层失效会先伤害什么结果

写法要求：

- 直接点名层级
- 指出判断依据
- 不用抽象术语堆砌

### 3. 关键缺陷

必须包含：

- 1 到 3 个最关键的问题
- 每个问题对应严重度
- 每个问题必须有文本依据或结构依据

写法要求：

- 只列最影响结果的问题
- 每个问题都要能落到具体失效点
- 不把小毛病堆成长列表

### 4. 影响解释

必须包含：

- 每个关键缺陷分别会伤害什么
- 影响路径必须具体，不得写成“会影响效果”

写法要求：

- 用“因为……所以……”说清机制
- 优先解释对进入、停留、信任、记忆、关注的影响
- 不引用平台黑箱作为主要解释

### 5. 修正优先级

必须包含：

- 先改什么
- 后改什么
- 为什么这个顺序不能反过来

写法要求：

- 按“上层优先于下层”的原则排序
- 不允许把表达问题排在结构问题前面
- 优先级必须和前面的严重度一致

### 6. 修改方向

必须包含：

- 面向每个关键问题的方向性修正建议
- 建议必须指向结构动作，而不是抽象评价

写法要求：

- 只给方向，不直接代写完整成稿
- 每条建议都要对应前文某个问题
- 能说清“补什么、删什么、前置什么、收窄什么、验证什么”

推荐动作词：

- 收窄承诺
- 前置核心问题
- 补过程证据
- 补机制解释
- 删去空转铺垫
- 降低结论强度
- 增加适用边界

禁止事项：

- 不直接给整篇重写稿
- 不给与问题无关的风格建议
- 不用“更高级一点”“更有感觉一点”之类空话

## 输出风格要求

输出必须满足以下风格约束：

- 直接
- 客观
- 结构优先
- 结论清楚
- 不做情绪安慰式表达
- 不堆砌方法论名词

## 长度控制

为避免输出失真，默认控制如下：

- 低语境初筛：总计 4 段，单段尽量 1 到 3 句
- 完整首轮：总体结论 1 句，关键缺陷 1 到 3 条，修正优先级最多 3 步，修改方向最多 3 条

## 低语境附加规则

当首轮任务属于低语境初筛时，输出必须明确：

- 当前结论以结构风险判断为主
- 不对平台分发、目标受众、商业效果做确定性结论

## 禁止输出的内容

以下内容不得出现在首轮输出中：

- 完整代写稿
- 多平台运营建议
- 账号总策略
- 夸张化评价
- 无依据的受众判断
- 只有结论没有归因的“拍板式点评”

## 输出前自检

输出前必须检查：

1. 是否正确选择了“低语境初筛”或“完整首轮”分流
2. 是否先给诊断，再给建议
3. 是否每条建议都能回指到一个问题
4. 是否区分了主问题和次问题
5. 是否避免了越界输出
