# Severity Rules

## 目的

本文件定义两套固定判定：

1. 文案状态等级
2. 单个问题严重度等级

两套判定必须同时存在。状态等级回答“这条文案总体处于什么状态”，严重度回答“同一层内部哪些问题先改，哪些后改”。

主问题层级与严重度不是同一个维度。主问题层级必须先按“最高失效层”锁定，再在该层内部判断严重度。

## 一、文案状态等级

### A 级

定义：

- 目标清晰
- 上层结构成立
- 既有进入能力，也有价值兑现能力
- 关键判断有证据或过程支撑
- 没有会直接破坏信任的硬伤

典型状态：

- 结构完整
- 有信息增量
- 有真实过程或可验证论证
- 结论有修正且边界诚实
- 表达层只剩局部优化项

### B 级

定义：

- 主结构成立
- 但存在一个关键薄弱点，已经明显影响传播或信任
- 不需要推倒重来，但必须先修关键缺口

典型触发：

- 有进入点，也有核心问题，但证据不够扎实
- 有过程，但机制解释不足
- 有反常识结论，但边界不够诚实
- 开头与正文基本一致，但承诺仍偏大

### C 级

定义：

- 局部有亮点
- 但核心矛盾没有处理好，整体不稳定
- 修词无法解决，需要回到目标、原则或结构重整

典型触发：

- 目标存在，但广度与深度严重失衡
- 有观点，但没有支撑观点的过程
- 节奏很快，但信息增量弱
- 反常识姿态存在，但不具备成立条件
- 表面完整，实际无法让用户复述新判断

### D 级

定义：

- 主要依赖表面包装或情绪推动
- 上层结构明显缺失
- 当前文本不适合继续微调，应先重构

典型触发：

- 没有清晰用户任务
- 只有钩子，没有兑现
- 没有过程、没有证据、没有机制
- 以强观点、强冲突替代真实深度
- 多个上层模块同时失效

## 二、状态等级的判定顺序

按以下顺序定级，不使用平均分：

1. 先看是否直接触发 D 级条件
2. 未触发 D 级，再看是否满足 C 级条件
3. 未触发 C 级，再看是否满足 B 级条件
4. 只有上层都基本成立时，才给 A 级

如果同时命中多个等级，取更低等级。

## 三、D 级直接触发条件

出现以下任一情况，可直接判定为 D 级：

- 目标层失效，用户任务不清晰
- 进入承诺与正文兑现严重错位
- 核心结论完全缺少可验证支撑
- 主要依赖情绪刺激、立场或包装成立
- 反常识结论没有机制解释，且没有边界
- 需要重构目标或结构，而不是局部修补

## 四、C 级核心判据

未触发 D 级，但出现以下任一情况，至少判为 C 级：

- 核心矛盾未处理，代价明显大于收益
- 原则层有两项及以上明显失效
- 结构层缺失关键证据或机制解释
- 信息增量、真实过程、反常识结论三项中，用户主诉项未成立
- 文案可看，但不能形成稳定判断

## 五、B 级核心判据

未触发 C 或 D，且满足以下特征时判为 B 级：

- 主目标清楚
- 主要结构模块已成立
- 至少有一个关键薄弱项正在拉低表现
- 该问题修正后，文案有机会进入 A 级

## 六、A 级核心判据

只有同时满足以下条件时，才可判为 A 级：

- 目标层通过
- 原则层没有明显失效项
- 结构层具备进入点、核心问题、关键证据或过程、机制解释
- 如果使用反常识结论，则其成立条件清楚
- 语境置信度允许作出稳定判断

## 七、问题严重度等级

### 致命问题

定义：

- 不解决，整条文案的主要逻辑无法成立
- 常见于目标层、原则层、结构层上游失效

典型触发：

- 没有用户任务
- 没有兑现
- 没有支撑核心结论的证据或过程
- 结论严重越界
- 伪反常识

处理要求：

- 必须优先解决
- 不得用表达优化替代

### 结构问题

定义：

- 不一定让整条文案彻底失效
- 但会显著伤害进入、停留、信任、记忆或关注中的至少一项

典型触发：

- 承诺过大
- 过程不够支撑结论
- 机制解释缺失
- 结构顺序错误
- 适用边界缺失

处理要求：

- 在致命问题解决后优先处理
- 必须给出结构性修改方向

### 表达问题

定义：

- 底层结构大体成立，但表达效率拖后腿

典型触发：

- 冗长
- 句子承载过满
- 抽象词过多
- 重复铺垫

处理要求：

- 不能单独作为主结论，除非上层已通过
- 修改建议应围绕压缩、拆分、去空话

### 优化项

定义：

- 不影响成立与否
- 只影响完成度和表现上限

典型触发：

- 个别措辞不够利落
- 个别顺序还能更顺
- 某个记忆点还能更集中

处理要求：

- 永远排在最后
- 不与结构问题混排

## 八、七层到严重度的映射规则

为避免同类问题在不同执行轮次里漂移，七层到严重度的默认映射如下：

- 目标层
  默认优先映射为致命问题；如果只是轻度偏斜但尚未破坏主任务，可降为结构问题。

- 约束层
  当问题会直接导致兑现失真、信任失真或根本无法成立时，映射为致命问题；其余大多数情况映射为结构问题。

- 矛盾层
  默认映射为结构问题；只有当取舍已经同时破坏进入和价值兑现时，才上升为致命问题。

- 原则层
  当承诺失真、论证不可验证、边界严重失实时，映射为致命问题；其余原则层失效默认映射为结构问题。

- 结构层
  默认映射为结构问题；如果缺失的是核心问题、兑现、关键证据或机制解释，且已经使主任务失效，可上升为致命问题。

- 表达层
  默认映射为表达问题；只有当表达问题已经严重阻断理解时，才上升为结构问题。

- 运行层
  当用户主目标涉及长期信任、关注或可沉淀性时，默认映射为结构问题；否则可作为优化项处理。若一次性刺激已经反向伤害当前主任务，也可上升为结构问题。

## 九、严重度与层级的对应优先级

当一个问题同时涉及多个层级时，按更高层级记严重度：

- 目标层问题优先高于结构层问题
- 约束层、矛盾层、原则层、结构层、运行层问题优先高于表达层问题
- 上层已失效时，下层同类问题不重复升格

例子：

- “开头太慢”如果本质是没有进入点，记为结构问题，不记为表达问题
- “说得很顺但很空”如果本质是没有信息增量，记为原则或结构问题，不记为表达问题

## 十、主问题选择规则

最终输出只允许有一个主问题层级，选择规则如下：

1. 先锁定最高失效层，不能让更下层问题反向覆盖更上层失效
2. 只在同一层内部，才比较严重度高低
3. 同层严重度相同，再选同时伤害“进入”和“价值兑现”的问题
4. 其余问题作为关键缺陷，不抢占主问题位置

## 十一、禁止事项

以下做法禁止出现：

- 给出等级但解释不了为什么
- 把多个小问题堆成“严重”
- 因为表达不顺，就把上层问题降级
- 因为有一个亮点，就掩盖致命问题
- 只给问题名称，不说明它伤害哪一项结果
