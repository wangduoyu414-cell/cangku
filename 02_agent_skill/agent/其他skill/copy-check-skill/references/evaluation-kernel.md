# Evaluation Kernel

## 目的

本文件定义本 skill 的固定判断顺序、各层的通过条件、失效信号、以及跨层归因规则。

本 skill 不采用“想到什么说什么”的点评方式。所有诊断都必须按层级推进，先判断上层是否成立，再判断下层如何修补。

## 固定顺序

七层校验顺序固定如下：

1. 目标层
2. 约束层
3. 矛盾层
4. 原则层
5. 结构层
6. 表达层
7. 运行层

不得跳过上层，直接在下层给解决方案。上层失效时，下层只可作为补充问题，不能被当作主问题。

## 记录方式

每层都必须留下四类记录：

- `状态`：通过、薄弱、失效
- `证据`：来自文本或输入的直接依据
- `影响`：它会伤害进入、停留、信任、记忆、关注中的哪一项
- `结论`：该层是否构成主问题

## 一、目标层

### 核心问题

这段文案是否同时服务于以下两件事：

- 让用户愿意进入
- 让用户看完后得到新的判断

### 通过条件

满足以下多数条件，可判为通过：

- 文案有明确的用户任务，而不是只有创作者表达欲
- 能看出主目标是什么，不是多个目标混杂
- 进入承诺和价值兑现方向基本一致
- 用户看完后能多得到一个明确判断，而不是只感受到情绪

### 失效信号

出现以下任一情况，目标层至少判为薄弱；出现多项则判失效：

- 只有吸引进入，没有价值兑现
- 只有观点表达，没有用户会改什么判断
- 一条文案同时承担引流、成交、说理、立人设等多个主任务
- 标题或开头承诺与正文兑现方向不一致
- 用户看完后只能复述态度，不能复述判断

### 主归因规则

以下问题优先归为目标层：

- “不知道这条到底想解决什么”
- “内容不算差，但主目标混乱”
- “有点开价值，没有看完价值”
- “只有结论，没有用户任务”

## 二、约束层

### 核心问题

文案是否承认现实约束，而不是假设用户会耐心、完整、无条件地理解它。

### 固定检查的四类约束

#### 1. 分发约束

检查文案有没有把核心价值压到过晚的位置。

通过信号：

- 前段就出现问题张力、异常结果或价值承诺
- 进入成本与文案时长、密度基本匹配

失效信号：

- 长铺垫后才进入真正问题
- 需要大量前置背景才能开始理解

#### 2. 注意力约束

检查文案有没有要求用户持续承担过高的解码成本。

通过信号：

- 核心问题清楚
- 句子负担不过重
- 逻辑推进连续

失效信号：

- 一句承载多个动作
- 抽象判断密集堆叠
- 读完一段仍不知重点

#### 3. 供给约束

检查文案是否建立在创作者可支撑的过程、经验或证据上。

通过信号：

- 关键结论有过程、比较、试错或样本支撑
- 结论强度与证据强度匹配

失效信号：

- 结论很重，证据很轻
- 用“我见过很多”“我研究很久”替代过程
- 借二手信息伪装一手经验

#### 4. 长期信任约束

检查文案是否为了短期刺激而伤害长期可信度。

通过信号：

- 有边界感
- 不把条件性判断讲成绝对规律
- 不靠夸张冲突制造全部价值

失效信号：

- 故意极端化
- 标题狠于正文太多
- 靠绝对化判断换取刺激

### 主归因规则

如果文案的主要问题是“假设用户会自动耐心理解”“假设作者天然有资格下结论”“假设夸张不会伤害信任”，优先归到约束层。

## 三、矛盾层

### 核心问题

文案有没有处理内容创作中的核心矛盾，而不是假装它们不存在。

### 必查四组矛盾

#### 1. 广度与深度

检查文案是在扩大入口，还是在加深判断，以及两者是否失衡。

失效信号：

- 入口非常泛，但正文没有窄兑现
- 内容极深，但入口重到无法进入
- 想同时讲所有人，导致没有一个问题讲透

#### 2. 爆发与信任

检查文案是在追求短期刺激，还是在积累长期预期。

失效信号：

- 靠强冲突、强站队、强情绪推动
- 没有可验证证据，只剩立场

#### 3. 节奏效率与论证完整

检查文案是否为了快，把必要论证删掉了。

失效信号：

- 结论前置后没有证据补足
- 过程被压缩到只剩口号
- 逻辑跳步严重

#### 4. 情绪刺激与判断质量

检查文案是否用情绪替代认知推进。

失效信号：

- 情绪强度高于信息密度
- 用户能感到态度，但拿不到新判断
- 评论性强，复述性弱

### 主归因规则

如果文案的主要问题是“不是不知道怎么说，而是不知道该牺牲哪一边”，优先归到矛盾层。

## 四、原则层

### 核心问题

文案是否满足五项底层原则。

### 原则一：入口低门槛

通过条件：

- 用户能在前段快速知道这条和自己有什么关系
- 进入成本低于预期收益
- 不依赖大量背景知识才能开始

失效信号：

- 开头泛铺垫
- 概念先行，问题后置
- 一开始就把复杂前提端上来

### 原则二：承诺高相关

通过条件：

- 承诺指向一个明确问题
- 承诺与正文兑现一致
- 承诺不是大而空的价值口号

失效信号：

- 承诺非常大，正文非常散
- 说要解决 A，正文其实在讲 B
- 用热词替代具体收益

### 原则三：论证可验证

通过条件：

- 核心判断有过程、对比、样本、失败路径、时间顺序中的至少一种支撑
- 证据能解释为什么结论成立

失效信号：

- 只有结论，没有过程
- 只有态度，没有证据
- 只有案例，没有机制

### 原则四：结论有修正

通过条件：

- 文案确实修正了某个常见判断
- 用户能复述“原来关键不是 X，而是 Y”

失效信号：

- 只是常识复述
- 只是换一种语气说旧观点
- 只给方法清单，没有认知修正

### 原则五：边界够诚实

通过条件：

- 结论强度与适用边界匹配
- 需要条件时会说明条件
- 暂不确定的部分会保留不确定性

失效信号：

- 绝对化结论
- 把个体经验讲成普遍规律
- 用“都”“一定”“根本”掩盖边界

### 主归因规则

如果文案的问题已经不是“顺不顺”，而是“底层原则站不住”，优先归到原则层。

## 五、结构层

### 核心问题

文案内部是否具备必要功能模块，并且顺序是否合理。

### 默认检查模块

#### 1. 进入点

要求：

- 有问题张力、异常结果、直接冲突或明确承诺中的至少一种

缺失信号：

- 开头只是背景、情绪、态度铺垫

#### 2. 核心问题

要求：

- 明确指出这条要解决的具体问题，而不是泛泛而谈

缺失信号：

- 说了很多，但不知道要打透什么

#### 3. 关键证据或过程

要求：

- 至少有一个支撑核心判断的过程片段
- 过程可以是试错、对比、筛选、时间顺序、失败样本、决策依据

缺失信号：

- 只有结果，没有得到结果的路径
- 只有“我做过”，没有“我怎么做”

#### 4. 机制解释

要求：

- 解释为什么这个结论成立
- 不是只给“是什么”，还要给“为什么”

缺失信号：

- 举了例子，但无法推出规律
- 只有现象描述，没有因果解释

#### 5. 反常识结论

要求：

- 结论修正的是因果、顺序或优先级，而不是单纯唱反调
- 反常识必须建立在前文证据和机制之上

缺失信号：

- 只有更激烈的表态
- 结论意外，但没有解释
- 结论对立，却不成立

#### 6. 适用边界

要求：

- 说明结论在什么条件下成立，或者至少避免绝对化表述

缺失信号：

- 把阶段性、条件性的判断讲成普遍定律

### 顺序规则

结构层默认推荐的展开顺序是：

1. 进入点
2. 核心问题
3. 关键证据或过程
4. 机制解释
5. 反常识结论
6. 适用边界

不是所有文案都必须显性写满六段，但这六种功能不能同时缺失。判断重点是“功能有没有承担”，不是“是否长得像模板”。

### 结构归因规则

- 缺进入点，优先影响进入
- 缺关键证据或过程，优先影响信任
- 缺机制解释，优先影响深度和复述性
- 缺反常识结论，优先影响记忆点和传播性
- 缺适用边界，优先影响长期可信度

## 六、表达层

### 核心问题

在上层基本成立的前提下，表达是否拖慢了理解效率。

### 重点检查项

- 是否冗长
- 是否一句承担过多任务
- 是否过度抽象
- 是否出现重复铺垫
- 是否用情绪加强代替逻辑推进

### 归因规则

表达层只处理“怎么更快被理解”，不处理“为什么说不成立”。

以下问题优先归为表达层：

- 同样意思绕了太多句
- 句子太满，读者难以抓主干
- 重复强调但没有新信息
- 概念堆砌导致理解变慢

如果目标层、原则层、结构层已失效，不得把表达层设为主问题。

## 七、运行层

### 核心问题

这段文案是否值得进入长期内容系统，而不是只能靠一次性刺激成立。

### 检查项

- 是否能让用户形成稳定预期
- 是否能沉淀为可复用的内容模式
- 是否具备后续扩展、复检、系列化的空间
- 是否可能因过度刺激而伤害长期信任

### 通过信号

- 价值承诺清晰且可复用
- 结论依赖的证据类型可以稳定提供
- 不是一次性情绪秀

### 失效信号

- 靠极端刺激成立
- 结论不可复验
- 一次有效，长期不可持续

## 跨层专检

以下三项必须在最终综合判断中单独过一遍。

### 信息增量专检

通过条件：

- 用户看完后能复述一个被修正的判断
- 这个修正不是常识复读，而是因果、顺序或重点的推进

失效信号：

- 只是把大家都知道的事讲顺了
- 只是把旧观点讲得更狠
- 看完只有态度，没有新判断

### 真实过程专检

通过条件：

- 文案出现至少一种可验证过程信号：
  - 时间顺序
  - 失败路径
  - 对比筛选
  - 决策依据
  - 样本边界

失效信号：

- 只有“我做过很多”
- 只有结果截图，没有判断过程
- 过程存在，但和结论无关

### 反常识结论专检

通过条件：

- 反的是错误因果、错误顺序或错误优先级
- 有机制解释
- 有边界条件

失效信号：

- 只是反着说
- 只有制造意外，没有解释成立条件
- 为了反常识而反常识

## 综合归因规则

最终归因必须遵守以下规则：

1. 主问题层级取“最高失效层”，不是“最容易改的层”
2. 下层问题不能覆盖上层失效
3. 同时出现多层问题时，优先报告对进入和价值兑现双重伤害最大的那一层
4. 如果信息增量、真实过程、反常识结论三项中有一项是用户主诉目标，则该项失效至少上升为结构问题
5. 不采用平均分，不用加总小问题掩盖大问题
