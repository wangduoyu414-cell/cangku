# Input Contract

## 目的

本文件定义本 skill 接收什么输入、如何判定输入是否足够、以及信息不足时如何继续工作。

输入契约的目标不是把用户挡在门外，而是让每次判断都知道自己建立在什么信息上。

## 输入结构

本 skill 的输入固定拆成三类：

1. 文案本体
2. 语境信息
3. 目标约束

三类输入中，文案本体是硬前提，另外两类决定判断置信度和结论边界。

## 字段定义

### 一、文案本体

必需字段：

- `copy_text`
  待校验的原始文本。可以是完整文案，也可以是标题、开头、主段、结尾等片段。

处理规则：

- 保留原有顺序，不擅自重组内容
- 保留关键断句和换行，因为它们可能影响节奏判断
- 如果用户提供多个版本，只把最新版本视为当前评估对象；旧版本只在复检模式下用于比较

### 二、语境信息

建议字段：

- `content_slot`
  文案所在位置，例如标题、开头、主段、结尾、完整口播稿、图文正文、评论区置顶等

- `platform`
  发布环境，例如短视频口播、图文帖、直播预告等

- `audience`
  目标对象。如果没有，禁止自行脑补人群画像

- `symptom`
  用户当前遇到的问题，例如“有播放但不涨粉”“能讲完但记不住”“看起来顺但很空”

处理规则：

- `content_slot` 缺失时，只能在文本中存在明显结构标记时做有限推断
- `platform` 缺失时，不引入平台专属规则，只做一般性的注意力与结构判断
- `audience` 缺失时，不输出细分人群判断
- `symptom` 缺失时，不对历史效果作因果性归因

### 三、目标约束

建议字段：

- `primary_goal`
  本次最优先要解决的目标，只允许一个主目标

- `secondary_goal`
  可选。只作为次级约束，不得与主目标并列争夺判断中心

可归一的常见主目标：

- 进入：更愿意点开或继续看
- 停留：更愿意看完
- 信任：更像真实经验，更可信
- 记忆：看完后记得住一个判断
- 关注：更愿意关注或追更

处理规则：

- 如果用户给出多个并列主目标，先标记“目标冲突风险”
- 如果用户给出的目标与文本承载能力明显不匹配，直接记为目标层问题
- 如果目标未给出，只能做条件性推断，不得伪装成既定目标

## 输入完整度分级

### L0：无效输入

条件：

- 没有 `copy_text`

处理：

- 不进入诊断
- 只指出缺少待校验文本

### L1：弱输入

条件：

- 只有 `copy_text`

处理：

- 只允许进入“低语境初筛”
- 只做初步结构风险判断
- 必须标记为低语境置信度
- 不做平台、受众、商业层面的确定性结论
- 不输出完整状态等级
- 不输出完整主问题层级
- 不输出完整修正优先级

### L2：部分输入

条件：

- `copy_text` + `content_slot`
- 或 `copy_text` + `primary_goal`

处理：

- 仍只允许进入“低语境初筛”
- 可以输出比 L1 更具体的结构风险提示
- 但需要显式指出缺少的关键字段
- 不进入完整七层校验
- 不输出完整状态等级
- 不输出完整修正优先级

### L3：可充分判断输入

条件：

- `copy_text` + `content_slot` + `primary_goal`

处理：

- 先把 `primary_goal` 归一为单一主目标
- 只有主目标不冲突时，才进入完整七层校验
- 可以给出完整的结构诊断和优先级判断
- 语境置信度至少为中

### L4：高置信度输入

条件：

- L3 基础上再补充 `platform`、`audience`、`symptom` 中至少两项

处理：

- 可以更稳定地解释影响路径
- 仍然不得超出文案诊断边界

## 语境置信度标记规则

本 skill 输出时必须给出语境置信度，规则如下：

- 高：输入达到 L4
- 中：输入达到 L3
- 低：输入只有 L1 或 L2

语境置信度影响结论边界，不直接替代文案等级。

## 输入归一流程

进入评估前，按以下顺序归一输入：

1. 锁定当前要评估的文本版本
2. 识别分析对象是整条文案还是单个片段
3. 归一主目标，只保留一个主目标
4. 标记语境完整度和语境置信度
5. 记录用户显式提出的限制条件，例如时长、风格、不得改动的内容

如果第 3 步无法把主目标归一成单一目标，则当前任务不能进入完整七层校验，只能停留在目标冲突判断和低语境初筛。

## 信息不足时的处理规则

### 缺少文案用途

可以继续，但只能停留在低语境初筛；必须避免使用位置强相关的判断，例如“这段开头不够快”之类的定性结论，除非文本本身就是明显的开头结构。

### 缺少主目标

允许根据文本最明显的意图做条件性判断，但只能停留在低语境初筛；必须用“如果你的主目标是……”来表述，不得直接写成“你的目标是……”。

### 缺少历史表现数据

禁止把问题归因为“平台不推”“用户不买账”等历史结果，只能解释结构风险和可能影响路径。

### 缺少受众与平台

禁止引入特定平台的技术口径，也禁止假设特定受众的知识背景。

## 输入冲突处理

如果输入内部存在冲突，按以下顺序处理：

1. 以用户显式说明为准
2. 以文本直接呈现的事实为次准
3. 无法消解时，记录为“输入冲突”，并在目标层或边界层指出

常见冲突包括：

- 说要做信任，但文本全是强情绪钩子
- 说要做低门槛传播，但文本需要大量前置背景
- 说要做反常识，但文本只有更激烈的口号

一旦出现主目标冲突，当前任务默认不进入完整七层校验，先输出“目标冲突风险 + 需要归一的主目标”。

## 低语境初筛允许输出

低语境初筛只允许输出以下内容：

- 当前属于低语境判断
- 已观察到的结构风险
- 缺少哪些信息会影响完整诊断
- 若补齐信息，下一步应进入什么诊断路径

低语境初筛不允许输出以下内容：

- 完整状态等级
- 完整主问题层级
- 完整关键缺陷列表
- 完整修正优先级
- 平台、受众、商业效果的确定性结论

## 禁止事项

以下做法禁止出现：

- 没有文本就开始诊断
- 缺少目标时擅自定义商业目标
- 缺少语境时输出具体平台策略
- 把多个目标平均处理，不指出冲突
- 用“可能”“大概”堆砌模糊判断，而不说明缺什么信息
